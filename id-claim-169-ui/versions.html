All Versions

Complete version history of Claim 169 — collaboratively maintained by the Working Group.

Tag Number
169
Data Format
CBOR Map
Token Type
CWT
Encoding
Base45
Compression
zlib / Brotli
Signature
ED25519 / ECC
  1. v1.2.1 Latest TBA

    Biometric standardization, dual compression, improved key discovery.

    • Brotli compression alongside zlib
    • COSE-based key discovery (JWKS deprecated)
    • Claims 16/17 deprecated — face moved to Claim 62 biometric object
    • iss claim requires fully qualified URI
    View details
  2. v1.2.0 Stable January 2026

    Collaborative Working Group release with expanded attribute registry.

    • Formal multi-stakeholder working group (UNHCR, GIZ, Tech5, MOSIP…)
    • 75+ attributes across demographic and biometric categories
    • Secondary language name & address fields (Attrs 19–20)
    • Location code, legal status, country of issuance (Attrs 21–23)
    View details
  3. v1.1.0 Legacy 2024

    Extended biometrics and multilingual demographic fields.

    • Palm print biometrics (Claims 63–64)
    • Voice biometric (Claim 65)
    • Secondary language fields (Claims 19–20)
    • zlib magic number detection for format disambiguation
    View details
  4. v1.0 Legacy 2023

    Initial release — foundational CBOR identity structure and encoding pipeline.

    • 18 demographic + 13 biometric claims defined
    • COSE_Sign1 with ED25519 / ECC signature support
    • Base45 encoding with zlib compression
    • JWKS-based .well-known key discovery
    View details

Unassigned claim number ranges may be used in closed ecosystems with mutual agreement between issuer and consumer. For globally consumable attributes, consultation with the Claim 169 Working Group is required.
